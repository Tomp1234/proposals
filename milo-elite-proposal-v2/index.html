We heard you · Day one of the POC

The work order issue you raised. Here's what it is and how we'll fix it.

On the call you flagged that Milo wasn't distinguishing between parent jobs and the sub-tasks within them. This is a known Salesforce data shape issue and one of the first things we'll apply when the POC kicks off — before any numbers go in front of you.

The fix · Day one of the POC

One rule added to Milo's knowledge base

We'll add a short instruction to Milo's understanding of your Salesforce data:

"work order" = parent job (parent_work_order_id IS NULL)
"child order" = sub-task within a job

Default to parent orders in all counts and reports.
Only include child orders when specifically asked.

This is a one-line configuration change — no development, no code, no data migration. Once added, every question about work orders will automatically filter correctly. The 1,456 overdue figure gets re-run immediately after.

This is exactly how Milo gets refined per customer during a POC. Any other data quirks like this get the same treatment — you flag it, we fix it in the same session, it's permanent.
